# IPA午前問題の図表画像を手動投入するワークフロー

## 目的と背景

AP Studyでは、IPA公式問題PDFに含まれる図表を推測で描き直さず、原資料から必要な部分だけを安全に取り込みます。Codexの作業環境では、IPA公式ページを確認できてもPDFファイルの取得がHTTP 403で遮断される場合があります。その場合に自動取得を迂回したり第三者サイトの画像を代用したりせず、ユーザーが公式PDFを開いて切り出した画像を受け渡す方式を使います。

この方式は、公式資料を確認できない状態で問題文、正解、画像又は出典URLを推測しないためのものです。PDFそのものやbase64データはリポジトリへ入れません。

## 手動で画像を用意する手順

1. IPA公式サイトから対象年度・期の午前問題PDFと公式解答を開く。
2. 年度、期、問番号が候補情報と一致することを確認する。
3. 対象問題の解答に必要な図表部分だけを、欠けや不要な別問題が入らない範囲で切り出す。
4. 矢印、線種、凡例、注記、数値、文字が原資料どおりで、拡大時にも判読できることを確認する。
5. 透過が不要ならWebPで保存する。WebPにすると判読性が落ちる場合はPNG、JPEG又はSVGも使用できる。
6. 下記ルールに従って `public/assets/ipa/ap/...` に配置する。
7. 問題文、全選択肢、正解、図表、公式URLを公式問題PDFと公式解答で再確認してから問題データに `assets` を追加する。
8. `npm run verify:questions`、`npm run build`、`npm run verify:dist` を実行する。

## 保存形式・配置場所・ファイル名

- 推奨形式: `.webp`
- 使用可能形式: `.webp`、`.png`、`.jpg`、`.jpeg`、`.svg`
- 配置形式: `public/assets/ipa/ap/{era-season}/am/qNNN-figure-1.webp`
- 1枚目: `qNNN-figure-1.webp`
- 2枚目以降: `qNNN-figure-2.webp`、`qNNN-figure-3.webp`
- `NNN` は3桁の午前問番号とする。
- `{era-season}` は `r04-autumn`、`r06-spring` のように令和年度と期を表す。

例:

```text
public/assets/ipa/ap/r04-autumn/am/q052-figure-1.webp
public/assets/ipa/ap/r06-spring/am/q023-figure-1.webp
```

画像は必要な図表だけを切り出し、500KB以下を目安にします。500KB超は検証警告、1MB超は検証エラーです。可読性を損なう過度な縮小や圧縮は行いません。

## 問題データの `assets` 記述例

```ts
assets: [
  {
    type: 'image',
    src: '/IPA_AP_STUDY/assets/ipa/ap/r04-autumn/am/q052-figure-1.webp',
    alt: '令和4年度 秋期 午前 問52の図',
    caption: '設問の図',
    sourceName: '情報処理推進機構（IPA） 応用情報技術者試験 令和4年度 秋期 午前 問52',
    sourceUrl: '公式PDF URL',
  },
],
```

`sourceUrl` には実際に照合したIPA公式HTTPS URLだけを設定します。`alt` は必須で、年度・期・問番号と図の内容を、正解を明かさない表現で記述します。

## `verify:questions` が確認する内容

画像を参照する問題について、次を検証します。`assets` を持たない既存問題はエラーになりません。

- `assets` が配列で、各要素の `type` が `image` であること。
- `src` と `alt` が空でないこと。
- `src` が `/IPA_AP_STUDY/assets/ipa/ap/` で始まること。
- 対応する実ファイルが `public/assets/ipa/ap/` 配下に存在すること。
- 拡張子が `.webp`、`.png`、`.jpg`、`.jpeg`、`.svg` のいずれかであること。
- 500KB超を警告し、1MB超をエラーにすること。
- `caption`、`sourceName`、`sourceUrl` が存在する場合は文字列であること。
- 既存の表データ検証を引き続き行うこと。

## PR本文に記載する内容

- 手動切り出しに使用したIPA公式問題PDFと公式解答の確認方法。
- 追加した問題ID、年度・期・問番号。
- 追加した画像パス、形式、ファイルサイズ。
- 図表が対象問題のものと一致し、欠けや改変がないことの確認結果。
- 問題文、選択肢、正解、`sourceUrl` の照合結果。
- `npm run verify:questions`、`npm run build`、`npm run verify:dist` の結果。
- 問題を追加しなかった場合は、画像未配置又は公式確認未完了などの理由。

## 画像付き問題を追加してはいけない条件

次のいずれかに該当する場合は、画像や問題データを追加せず、[図表問題候補リスト](./ipa-figure-question-candidates.md) の状態とメモを更新します。

- IPA公式問題PDF又は公式解答を確認できない。
- 画像ファイルが配置されていない、又は対象問題の図表だと確認できない。
- 問番号、切り出し範囲、図表の欠け、文字、線、凡例、注記のいずれかが曖昧である。
- 問題文、選択肢、正解、解説、出典URLの一部でも推測が必要である。
- AI生成画像、描き直した推測画像、第三者サイト由来の画像しか用意できない。
- PDF丸ごと、base64埋め込み、又は1MBを超える巨大画像になる。
- 適切な `alt` を正解のネタバレなしに記述できない。
- 午後問題である。
