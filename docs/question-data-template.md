# 問題データ追加ガイド

問題は `src/data/questions.ts` の `questions` 配列へ追加します。既存の `createQuestion` を利用し、問題文・選択肢・正答・解説・出典を1問単位で記述してください。大量追加は小さな単位に分け、追加ごとに検証します。

## ID命名ルール

IDは `ap-r05-autumn-am-q001` 形式です。

| 要素 | 値の例 | 意味 |
| --- | --- | --- |
| 試験区分 | `ap` | 応用情報技術者試験 |
| 年度 | `r05`, `r06` | 令和年度（2桁） |
| 季節 | `spring`, `autumn` | 春期、秋期 |
| 区分 | `am`, `pm` | 午前、午後 |
| 問番号 | `q001` | 3桁の問番号 |

既存IDを変更する場合だけ、回答履歴・ブックマーク・進行中セッションを引き継ぐため、変更前の値を `legacyId` に指定します。新規問題では `legacyId` を省略し、データ作成時の一時IDではなく最初から正式IDを設定してください。

## field一覧

`field` は `src/data/fields.ts` の `FIELDS` から選択します。文字列を直接追加せず、次の既定分野を使用してください。

- 基礎理論
- コンピュータシステム
- 情報セキュリティ
- ネットワーク
- データベース
- システム開発
- プロジェクトマネジメント
- サービスマネジメント
- システム監査
- ストラテジ

新しい分野が必要な場合は、先に `FIELDS` へ追加し、表記を一元管理します。

## explanationの書き方

- `correctReason`: 正答になる理由を、根拠が分かる文章で記載します。
- `wrongReasons`: 誤答となる各選択肢の理由を、選択肢キー（`ア`、`イ`、`ウ`、`エ`）ごとに記載します。
- `points`: 解法や暗記の要点を文字列配列で記載します。
- `keywords`: 復習に使える主要語を文字列配列で記載します。
- 公式解説を転載せず、AP Study独自の説明として簡潔に記載します。

## 出典表示ルール

各問題に `sourceName`、`sourceUrl`、`isQuoteFromIpa` が必要です。現在は `createQuestion` が出典情報を設定します。

- IPA公開問題の問題文・選択肢を引用する場合は `quoted: true` とし、対象問題に合うIPA公開ページまたはPDFを出典にします。
- オリジナル問題では `isQuoteFromIpa: false` とし、参照したIPAシラバスなどを示します。
- 問題文・選択肢の出典と、AP Study独自解説を混同しない表現にします。

## 追加後の確認

```bash
npm install
npm run verify:questions
npm run build
npm run verify:dist
```

検証エラーが出た場合は、表示された問題IDと項目を修正してからコミットしてください。
